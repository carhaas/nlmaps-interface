What is a reference point?

A reference point is an area in which you want to search. It can be a city, a state or a country. If you supply a reference point manually, you can help guide the system to search in the appropriate area. For example, if you live in Heidelberg, Pennsylvania but only use Heidelberg in your question, the system will assume you refer to Heidelberg, Germany. By typing 'Heidelberg, Pennsylvania' into the reference point text box, you can ensure that the system finds the location you are interested in. Of course, an even faster option is to click 'Use my location' if you want search results about the city you are currently in.

Please note: If you supply a reference point or clicked on 'Use my location', you should not mention a reference point again in your question!

Is there anything special about Heidelberg, Paris and Edinburgh?

You might have noticed that Heidelberg, Paris and Edinburgh feature prominently in our example questions. This is because the system was largely trained with questions about these 3 cities. Consequently these are the cities with which the system works the best. However, other cities also work.

The results don't make any sense, why is that?

Because the system is still under development, you may sometimes observe results that don't make any sense. For example you might ask Where something is and instead it counts the number of objects. Please bear with us, these kinks will iron themselves out as as the system learns. In fact, you can help us improve by letting us know about such weird answers by clicking the 'No' button when we ask you 'Was that helpful?'

Some objects have less information than others, why is that?

Maybe you have seen some objects (e.g. restaurants) that have a telephone number listed and others that do not. This is because we can only display information that is present in the OpenStreetMap database.
